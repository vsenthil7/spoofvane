About this guide

This guide is organised by task. For each one you get: who can do it, what you need first, numbered steps, the screen shown beside the steps with matching ● markers, the result, and what to do next.

Your role decides what you can do

ViewerRead dashboards and settings.
Auditor+ Read audit & compliance.
Reviewer+ Approve/deny HITL actions (≠ analyst).
Analyst+ Triage, investigate, request takedowns, onboard brands.
Admin+ Manage users, agents, cost, tenants.
Owner+ Manage tenants and billing.
The golden rule: every takedown and sensitive action needs a second person's approval — you propose, a reviewer approves — and everything is recorded in a tamper-evident audit log.

Getting started

T1

Sign in and find your way around

Access SpoofVane and learn the layout.  Who: All roles

Before you start
  • An account from your administrator
  • SSO, or email + password
  • An authenticator app if MFA is on
Steps
  1. Open the console. Go to your SpoofVane URL — the Login screen appears.
  2. Choose a sign-in method. Click SSO (OIDC) or SAML, or enter email + password.
  3. Complete MFA. Enter the 6-digit code if prompted.
  4. Orient yourself. You land on the Dashboard; the left rail is your nav.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

You're signed in on the Dashboard with today's KPIs and live pipeline.

→ Next

Analysts go to Triage (T4); to add a brand first see Onboard a brand (T2).

Behind the scenes: Identity (G1) handles SSO/MFA; RBAC (G2) controls what you see.

Set up protection

T2

Onboard a brand to monitor

Register a brand so discovery starts finding impersonation.  Who: Analyst, Admin, Owner

Before you start
  • Signed in
  • The brand's canonical login URL
  • (Optional) a reference logo
Steps
  1. Open Brands. Click Brands in the left rail.
  2. Launch the wizard. Click + New brand (wizard).
  3. Enter basics. Brand name + login URL; accept the suggested keywords, TLDs and regions.
  4. Set threshold. Pick the escalation score threshold (default 0.65).
  5. Save. You're taken to the Brand detail screen.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

The brand is live; within minutes the Triage queue fills with suspects.

→ Next

Watch results in Triage (T4).

Behind the scenes: Brand Wizard is H3; saving activates Discovery (A1–A10) over Bright Data SERP, Proxies & Datasets.
T3

Protect an executive from deepfakes

Monitor an executive's likeness and voice.  Who: Analyst (with lawful basis), Admin

Before you start
  • A brand is onboarded
  • A named lawful basis
  • (Recommended) a reference voice clip
Steps
  1. Open Exec protection. Click Exec protection in the rail.
  2. Add the executive. Enter the name and platforms to watch.
  3. Record the lawful basis. Required — monitoring will not start until you enter it.
  4. Enrol a voiceprint. Optionally upload a clean voice sample for cloned-voice matching.
  5. Activate. Confirm to begin scanning.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

Impersonating handles, likeness/voice assets and deepfake media are reported with a risk score.

→ Next

Suspect media appears in Deepfakes (T6).

Behind the scenes: Exec surface is H5 (sensitive → governance gate E8, §V8-5); voiceprint C9; deepfake C10.

Investigate

T4

Triage the alert queue

Work through suspects and pick what needs action.  Who: Analyst

Before you start
  • A brand has produced suspects
Steps
  1. Open Triage queue. Each row is a suspect: verdict pill, score, source, SLA timer.
  2. Sort by urgency. Red phish pills and short SLA timers come first.
  3. Open a suspect. Click a row to open Alert detail.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

The most urgent suspect is open, ready for evidence review.

→ Next

Review the evidence — Review an alert (T5).

Behind the scenes: Ordering uses the calibrated probability C1 (not a raw sum), from Scoring C1–C8.
T5

Review an alert and confirm the verdict

Decide if a suspect is genuinely malicious using the full evidence.  Who: Analyst

Before you start
  • A suspect open in Alert detail (T4)
Steps
  1. Read Summary. Verdict, calibrated score, brand, first-seen.
  2. Inspect evidence. Open Evidence, Multi-region (cloaking?) and Kit (which kit matched).
  3. Check Verdict trace. See the Claude+GPT+Gemini ensemble; dissent discounts confidence & flags for review.
  4. Review context. MITRE techniques, Cluster links, Deepfake signals.
  5. Decide. Click Request takedown, or mark benign to dismiss.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

Your decision is recorded; a takedown is queued for approval, a dismissal closes the alert.

→ Next

A reviewer approves the takedown next — Approve a takedown (T8).

Behind the scenes: Ensemble D1–D5, dissent D5, MITRE D7, clustering C8; your call feeds active-learning D8/E6.
T6

Review a suspected deepfake

Confirm or dismiss AI-generated impersonation media.  Who: Analyst

Before you start
  • A deepfake suspect exists (from T3 or discovery)
Steps
  1. Open Deepfakes. Each item shows the multimodal score.
  2. Read the signals. Face distance, lip-sync, voiceprint match, C2PA provenance — valid provenance lowers suspicion.
  3. Confirm. Accept or override the multimodal verdict.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

The media is classified with per-signal reasoning kept for evidence.

→ Next

Request a takedown (→ T8); legal export carries a chain-of-custody pack.

Behind the scenes: Voiceprint C9, ensemble C10, fusion D6, legal custody pack G10.
T7

See the campaign behind a site

Find whether a suspect is part of a larger campaign.  Who: Analyst

Before you start
  • Two or more related suspects exist
Steps
  1. Open Clusters. See the threat-network graph.
  2. Read the links. Nodes connect by shared ASN, registrar, kit, favicon or TLS fingerprint.
  3. Jump to a node. Click any node to open that site's Alert detail.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

You see the whole campaign and can act on it as a group.

→ Next

Take the sites down individually (each via T5 → T8).

Behind the scenes: Clustering C8 (union-find); graph maintained by cluster agent E5.

Act

T8

Approve or deny a takedown (reviewer)

Provide the mandatory human approval before submission.  Who: Reviewer (≠ the analyst)

Before you start
  • An analyst requested a takedown (T5)
  • You hold the Reviewer role
Steps
  1. Open Review queue. Pending actions are listed.
  2. Examine the request. Review the suspect, verdict and drafted packet.
  3. Approve or deny. Approve to authorise; deny with a reason.
  4. Second authoriser. Egress actions (canary creds / honeypot) need a second approver.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

On approval the takedown is submitted and a reference ID returns; all audited.

→ Next

Track delivery in Takedowns (T9).

Behind the scenes: HITL G4 (reviewer ≠ analyst); two-person rule via governance E8 (§V8-5).
T9

Track takedown delivery

Monitor where each approved takedown sits in the pipeline.  Who: Analyst, Reviewer

Before you start
  • A takedown was approved (T8)
Steps
  1. Open Takedowns. See URL, channel, status and reference ID per row.
  2. Read the status. Registrar or hosting-abuse channel and its state.
  3. Override if stalled. Re-route or escalate manually.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

Live visibility of every takedown and its reference.

→ Next

Closed alerts remain in Audit (T10) permanently.

Behind the scenes: Registrar adapters F1–F3, hosting-abuse F4; alerts also fan out to SIEM/SOAR via F5.

Oversee

T10

Search the audit trail

Find who did what, when — for investigation or audit.  Who: Auditor, Reviewer, Admin

Before you start
  • A role with audit access
Steps
  1. Open Audit. The hash-chained log opens.
  2. Ask in plain language. Type e.g. 'analyst takedown actions for tenant acme'.
  3. Read the chained log. Actor, action, outcome and a hash linking each entry.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

A filtered, tamper-evident view of the relevant actions.

→ Next

For reporting, export an auditor pack from Compliance (P14).

Behind the scenes: NL search H2; hash-chained audit G3 self-verifies and detects tampering.

Watch Bright Data spend

Before you start
  • Admin or Owner role
Steps
  1. Open Cost. See spend vs the tier cap.
  2. Read the bar. Current spend, envelope, and SLM cheap-path share.
  3. Set alerts. Configure a budget alert threshold.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

You can see and control cost before it exceeds the envelope.

→ Next

If high, review active sources in Brand detail (P04).

Behind the scenes: Cost tracking G9; every BD call records a cost row; SLM-first routing D4 keeps costs down.

Administer

T12

Manage agents and the kill-switch

Govern autonomous agents and halt them instantly if needed.  Who: Admin

Before you start
  • Admin role
Steps
  1. Open Admin · Agents. See each agent, its state and blast-radius budget.
  2. Set budgets. Adjust the cost/risk envelope per agent.
  3. Use the kill-switch. Global to stop everything, or per-tenant to scope it.
The numbered markers ● on the screen match the steps.
✓ Result — what happens

Agents run within budget; firing the kill-switch halts them all (audited).

→ Next

Resume agents from the same screen when safe.

Behind the scenes: Kill-switch E7, governance E8, hash-chained agent audit E9.
