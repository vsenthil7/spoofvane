How this guide works

SpoofVane is a Security-Operations-Center (SOC) console for brand, executive, deepfake and fraud-surface defense. Every authenticated screen renders inside a fixed left sidebar (grouped into Operations, Settings and Admin) and a scrollable content pane, in a dark-first SOC theme. For each screen this guide gives you What it is · Key elements · Actions · Leads to →, beside a capture of the screen. Part 2 is the screen-by-screen tour; Part 3 is the end-to-end flows; Part 4 covers roles, page states and exports.

Part 1 · Screen-by-screen

1

Dashboard

/

What it is. The SOC at-a-glance landing screen after sign-in.

Key elements
  • KPI tiles: Total Alerts, Open Alerts, Deepfakes (24h), Active Clusters, Exec Protection, Takedown Rate
  • Alert-trend chart, Top Targeted Brands, Live Pipeline strip, Attack-Family distribution
  • Capability Status (agents) and Compliance Health tiles
Actions

Click a KPI tile or chart segment to jump into the filtered view behind it.

Leads to → Triage Queue · Deepfake Feed · Campaign Clusters · Exec Protection · Cost Attribution
2

Triage Queue

/triage

What it is. The analyst's primary work surface — alerts awaiting a decision.

Key elements
  • Virtualised alert table: severity, brand, suspect URL, family, score, SLA timer, status
  • Filters, saved views, bulk-select
  • CSV / NDJSON export
Actions

Open an alert, assign it, change status, bulk-action, export the queue.

Leads to → Alert Detail (click any row)
3

Alert Detail

/alerts/:id

What it is. The full evidence dossier for one alert; where verdicts and takedowns are actioned.

Key elements
  • Header: family, assignee, threat flags, scoring breakdown, kit name/version/confidence
  • 8 evidence tabs: Overview, Evidence, Verdict, Multi-region (cloaking), MITRE, Ledger, Cluster, Takedown
  • Every consequential action writes a hash-chained audit entry
Actions

Confirm / deny the verdict, draft & submit a takedown (behind a human-in-the-loop gate), add notes.

Leads to → Campaign Clusters (Cluster tab) · Audit Log (Ledger tab) · Review Queue (on takedown)
4

Campaign Clusters

/clusters

What it is. See related threats as a network — kits / domains / hosting / actors in one campaign.

Key elements
  • Interactive threat-network graph (shared hosting ASN, kit family, registrar, favicon, TLS fingerprint)
  • Cluster-member side panel with attacker profile and recommended actions
Actions

Select a node to inspect; pivot from a member into its Alert Detail.

Leads to → Alert Detail
5

Deepfake Feed

/deepfakes

What it is. Review queued deepfake media (image / audio / video) with model verdicts.

Key elements
  • Queue of clips/images with face-embedding, lip-sync, voiceprint and C2PA-provenance scores
  • Per-item verdict panel and evidence pack
Actions

Confirm / deny a deepfake verdict; open the chain-of-custody pack.

Leads to → Alert Detail (when tied to an alert)
6

Exec Protection

/exec-protection

What it is. Monitor executive likeness and voice abuse across the open web.

Key elements
  • Per-executive surface findings (likeness hits, voice exposure from OSINT crawlers)
  • PII leakage score, data-broker exposure, redaction task list
Actions

Review a finding, trigger a redaction recommendation, escalate (PII processing is lawful-basis gated).

Leads to → Alert Detail · Audit Log
7

Brand Management

/brands

What it is. Manage the brands SpoofVane protects.

Key elements
  • Brand list with protected regions, canonical assets and detection thresholds
  • Active discovery channels per brand
  • Brand-onboarding wizard launcher
Actions

Add a brand, launch the wizard, edit thresholds.

Leads to → Brand Detail · Dashboard (post-onboarding)
8

Cost Attribution

/cost

What it is. Per-tenant spend on Bright Data products and LLM / SLM calls.

Key elements
  • Spend breakdown by product / action / tenant, budget bars, alerting thresholds
  • Bright Data rows tagged source=brd_*; SLM-first routing share
Actions

Set budgets; export the cost-proof report (rules in docs/COST_ENVELOPE.md).

Leads to → Compliance · Audit Log
9

Audit Log

/audit

What it is. The tamper-evident record of every consequential action.

Key elements
  • Searchable hash-chained trail (Blake2b + Ed25519 signed), 7-year retention
  • Natural-language search
  • Chain verification + audit-pack export
Actions

Search, verify the chain, export the audit pack.

Leads to → Alert Detail (each entry links back to its subject)
10

Campaign — Agent Control

/admin/agents

What it is. The control room for the autonomous agents (J1–J10).

Key elements
  • Agent registry with status and governance budgets
  • Global Kill-Switch (propagates in ≤ 5 s)
  • Governance framework banner (NIST AI RMF + EU AI Act + UK ECCTA)
Actions

Start / pause an agent, set budget / risk envelopes, stop every agent immediately.

Leads to → Audit Log (every action recorded)

Compliance

Key elements
  • Framework tiles: SOC 2, ISO 27001, GDPR, NIST AI RMF, EU AI Act, ECCTA
  • Per-framework control mappings + evidence-pack export

Export a signed compliance pack; drill into a control's evidence.

Leads to → The compliance/ document pack

Tenants

Leads to → Users (RBAC)

Demo Health

/admin/demo-health

What it is. Internal readiness check — confirms the seed populates every page and acceptance criteria pass.

Takedowns

/takedowns

What it is. Multi-channel takedown status across registrars and hosting-abuse channels.

Key elements
  • Per-takedown channel (Cloudflare / GoDaddy / Namecheap / hosting abuse), status, reference ID
  • Manual override on stalled channels
Actions

Track delivery; re-route or escalate a stalled takedown.

Leads to → Audit Log
15

Review Queue

/review

What it is. The human-in-the-loop approval surface (reviewer ≠ analyst enforced).

Key elements
  • Pending takedowns/verdicts with the drafted packet and full verdict trace
  • Second-authoriser step for egress actions (canary creds / honeypot)
Actions

Approve or deny; provide a second authorisation where required.

Leads to → Takedowns (on approval) · Audit Log

Part 2 · Screen flows

A — Daily triage (the core loop)

Dashboard → Open Alerts KPI → Triage Queue (filter, pick an alert) → Alert Detail → Evidence tab (screenshots, DOM, multi-region cloaking) → Verdict tab (confirm/deny) → Takedown tab (draft) → Review Queue (HITL approve) → Takedowns (track) → Audit Log (hash-chained) → back to the queue.

B — Brand onboarding

Brand Management → Add brand / launch wizard → enter brand + regions + thresholds → wizard provisions discovery (A1–A10) → Dashboard begins to populate alerts → Brand Detail.

C — Deepfake review

Dashboard Deepfakes KPI → Deepfake Feed → open a clip (face / voiceprint / lip-sync / C2PA scores) → confirm/deny → chain-of-custody pack → Alert Detail (if tied to an alert).

D — Executive protection

Exec Protection → pick an executive → review likeness / voice findings → record lawful basis → action a redaction recommendation → escalate → Alert Detail / Audit Log.

E — Campaign investigation

Triage / Dashboard → Alert Detail → Cluster tab → Campaign Clusters graph → pivot across kit / hosting / actor nodes → open related Alert Details.

F — Agent governance & kill-switch

Admin › Agent Control → inspect agent status + budgets → (incident) press the Global Kill-Switch → all agents halt ≤ 5 s → Audit Log confirms with a hash-chained entry.

G — Cost & compliance review

Cost Attribution → review per-tenant Bright Data + LLM spend, set budgets → Admin › Compliance → check framework status → export signed evidence pack.

H — Audit investigation

Audit Log → NL search (“reviewer overrides last week”) → open a result → Alert Detail (the subject) → verify hash-chain integrity.

Part 3 · Roles, states & exports

Roles & visibility
ViewerRead dashboards and settings.
Auditor+ Read the audit log and compliance evidence.
Reviewer+ Approve / deny HITL actions; reviewer ≠ analyst is enforced.
Analyst+ Triage, investigate, request takedowns, onboard brands.
Admin+ Manage users, agents, cost, tenants, integrations.
Owner+ Manage tenants and billing.

Forbidden actions are hidden; a forbidden deep link shows the Forbidden (403) screen.

Page states & exports. Every data screen renders one of four states — Populated, Empty, Loading (skeletons), Error (with retry). Tables export CSV / NDJSON; alerts export a signed PDF; Compliance exports a signed evidence pack; deepfake evidence exports a chain-of-custody bundle.
