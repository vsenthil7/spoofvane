How this guide works

SpoofVane is a Security-Operations-Center (SOC) console for brand, executive, deepfake and fraud-surface defense. Each screen is shown with numbered ● markers placed on the elements you act on; the matching numbered steps tell you what to do. You also get What it is · Key elements · Actions · Leads to →. Part 2 is the end-to-end flows; Part 3 covers roles, page states and exports. Where a screen needs more than one page, the lower half is marked (continued).

Part 1 · Screen-by-screen

1

Sign in

/login

What it is. Access SpoofVane; the entry screen with SSO / SAML / local + MFA.  Who: All roles

● The numbered markers on the screen match the numbered steps below.

1

Sign in (continued)

Steps
  1. Click SSO (OIDC) or SAML to use your identity provider.
  2. Or type your email + password.
  3. Click Sign in.
  4. Enter the 6-digit MFA code if prompted — then you land on the Dashboard.
Key elements
  • SSO (OIDC) and SAML buttons
  • Email + password fallback
  • TOTP MFA challenge when enabled
Actions

Authenticate and reach the SOC console.

Leads to → Dashboard

Brand Management

/brands

What it is. Register and manage the brands SpoofVane protects; launch the onboarding wizard.  Who: Analyst, Admin, Owner

● The numbered markers on the screen match the numbered steps below.

2

Brand Management (continued)

Steps
  1. Click + New brand (wizard) to start onboarding.
  2. Enter the brand name and canonical login URL; accept the suggested keywords, TLDs and regions.
  3. Pick the escalation score threshold (default 0.65), then save to open Brand detail.
Key elements
  • Brand list with regions, canonical assets, detection thresholds
  • Active discovery channels per brand
  • Brand-onboarding wizard launcher
Actions

Add a brand, launch the wizard, edit thresholds.

Leads to → Brand Detail · Dashboard (post-onboarding)
3

Exec Protection

/exec-protection

What it is. Monitor an executive's likeness and voice abuse across the open web (sensitive surface).  Who: Analyst (lawful basis), Admin

● The numbered markers on the screen match the numbered steps below.

3

Exec Protection (continued)

Steps
  1. Enter the executive's name and the platforms to watch.
  2. Record a named lawful basis — required; monitoring will not start without it.
  3. Optionally upload a reference voice clip for cloned-voice matching.
  4. Click Activate scanning to begin.
Key elements
  • Per-executive findings: likeness hits, voice exposure, PII leakage score
  • Data-broker exposure + redaction task list
  • Lawful-basis gate before any monitoring
Actions

Review findings, trigger redaction, escalate (PII processing is gated).

Leads to → Alert Detail · Audit Log
4

Triage Queue

/triage

What it is. The analyst's primary work surface — alerts awaiting a decision.  Who: Analyst

● The numbered markers on the screen match the numbered steps below.

4

Triage Queue (continued)

Steps
  1. Scan the queue — each row is a suspect with its verdict pill, score, source and SLA timer.
  2. Work top-down: red phish pills and short SLA timers are most urgent.
  3. Click a row to open its Alert detail.
Key elements
  • Virtualised table: severity, URL, family, score, SLA timer, status
  • Filters, saved views, bulk-select
  • CSV / NDJSON export
Actions

Open, assign, change status, bulk-action, export.

Leads to → Alert Detail (click any row)
5

Alert Detail

/alerts/:id

What it is. The full evidence dossier for one alert; where verdicts and takedowns are actioned.  Who: Analyst

● The numbered markers on the screen match the numbered steps below.

5

Alert Detail (continued)

Steps
  1. Read the Summary tab — verdict, calibrated score, brand, first-seen.
  2. Open Evidence + Multi-region (cloaking) tabs to see the captured page.
  3. Open Verdict trace — the Claude+GPT+Gemini ensemble; dissent discounts confidence.
  4. Check MITRE / Kit / Cluster tabs for context.
  5. Click Request takedown (or mark benign to dismiss).
Key elements
  • Header: family, threat flags, scoring breakdown, kit name/version
  • Tabs: Summary, Evidence, Multi-region, Verdict, MITRE, Kit, Cluster, Deepfake, Takedown
  • Every action writes a hash-chained audit entry
Actions

Confirm / deny the verdict, draft & submit a takedown (HITL-gated), add notes.

Leads to → Campaign Clusters · Audit Log · Review Queue (on takedown)

Review Queue (continued)

Steps
  1. Open the queue — pending actions are listed.

Part 2 · Screen flows

A — Daily triage (the core loop)

Dashboard → Triage Queue → Alert Detail → Evidence/Verdict tabs → Request takedown → Review Queue (HITL) → Takedowns → Audit Log → back to the queue.

B — Brand onboarding

Brand Management → wizard → brand + regions + thresholds → discovery (A1–A10) provisions → Dashboard populates → Brand Detail.

C — Deepfake review

Dashboard Deepfakes KPI → Deepfake Feed → open a clip → confirm/deny → chain-of-custody pack → Alert Detail.

D — Executive protection

Exec Protection → pick executive → record lawful basis → review findings → redaction → escalate → Alert Detail / Audit.

E — Campaign investigation

Alert Detail → Cluster tab → Campaign Clusters graph → pivot across kit/hosting/actor → related Alert Details.

F — Agent governance & kill-switch

Agent Control → inspect status + budgets → (incident) Global Kill-Switch → all agents halt ≤ 5 s → Audit Log entry.

G — Cost & compliance

Cost Attribution → spend + budgets → Compliance → framework status → export signed evidence pack.

H — Audit investigation

Audit Log → NL search → open a result → Alert Detail → verify hash-chain integrity.

Part 3 · Roles, states & exports

Roles & visibility
ViewerRead dashboards and settings.
Auditor+ Read audit & compliance evidence.
Reviewer+ Approve / deny HITL actions; reviewer ≠ analyst enforced.
Analyst+ Triage, investigate, request takedowns, onboard brands.
Admin+ Manage users, agents, cost, tenants, integrations.
Owner+ Manage tenants and billing.

Forbidden actions are hidden; a forbidden deep link shows the Forbidden (403) screen.

Page states & exports. Every data screen renders one of four states — Populated, Empty, Loading (skeletons), Error (with retry). Tables export CSV / NDJSON; alerts export a signed PDF; Compliance exports a signed evidence pack; deepfake evidence exports a chain-of-custody bundle.
